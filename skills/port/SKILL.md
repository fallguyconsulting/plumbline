# /plumbline:port

Assess the current project's adoption state and emit a Plumbline porting plan with the project's actual per-check baseline numbers. The output is markdown — drop it into your planner of choice as the structural input.

For the full reference on each phase, see `docs/plumbline-porting-guide.md` in the plumbline plugin distribution (also linked at the top of the emitted plan).

## What this does

1. Runs `/plumbline:doctor` to identify what's already in place (plugin enabled, cheatsheet committed, config present, budget baseline saved).
2. Runs a temporary all-checks-on lint (atomic enable → audit → restore so `.plumbline.json` is not modified) to count violations per check.
3. Emits a phase-by-phase plan template with the project's per-check numbers substituted in, sequencing the five phases (Adopt, Audit, Sweep, Ratchet, Maintain) appropriate to the current state.

## Run

```bash
plumbline_bin="${CLAUDE_PLUGIN_ROOT%/}/bin/plumbline"
guide_path="${CLAUDE_PLUGIN_ROOT%/}/docs/plumbline-porting-guide.md"
target="${1:-.}"

echo "# Plumbline adoption plan — $(basename "$(cd "$target" && pwd)")"
echo ""
echo "Generated by /plumbline:port. Reference: $guide_path"
echo ""

echo "## Current state"
echo ""
node "$plumbline_bin" doctor "$target" 2>&1 | sed 's/^/  /'
echo ""

# Temporary all-checks-on audit; trap restores .plumbline.json on any exit path
cfg="$target/.plumbline.json"
restore_cfg() {
  if [ -f /tmp/plumbline-port-cfg.bak ]; then
    cp /tmp/plumbline-port-cfg.bak "$cfg" 2>/dev/null || true
    rm -f /tmp/plumbline-port-cfg.bak
  fi
}
trap restore_cfg EXIT INT TERM
if [ -f "$cfg" ]; then
  cp "$cfg" /tmp/plumbline-port-cfg.bak
  sed -i '' 's/"comment_hygiene": false/"comment_hygiene": true/' "$cfg" 2>/dev/null || true
  sed -i '' 's/"source_validity": false/"source_validity": true/' "$cfg" 2>/dev/null || true
  sed -i '' 's/"blessed_invariant_test_coverage": false/"blessed_invariant_test_coverage": true/' "$cfg" 2>/dev/null || true
fi
audit_out=$(node "$plumbline_bin" "$target" 2>&1 || true)
restore_cfg
trap - EXIT INT TERM

source_count=$(echo "$audit_out" | grep -c "plumbline/source-missing-file" || true)
symbol_count=$(echo "$audit_out" | grep -c "plumbline/source-missing-symbol" || true)
invariant_count=$(echo "$audit_out" | grep -c "plumbline/blessed-invariant-uncovered" || true)
hygiene_count=$(echo "$audit_out" | grep -c "plumbline/comment-hygiene" || true)
total=$((source_count + symbol_count + invariant_count + hygiene_count))

echo "## Backlog (with all three checks enabled)"
echo ""
echo "| Check | Count | Recommended approach |"
echo "| --- | --- | --- |"
echo "| source-missing-file | $source_count | $([ $source_count -lt 200 ] && echo 'Sweep (Phase 2a)' || echo 'Sweep in waves') |"
echo "| source-missing-symbol | $symbol_count | $([ $symbol_count -lt 50 ] && echo 'Sweep (Phase 2a)' || echo 'Triage per-symbol') |"
echo "| blessed-invariant-uncovered | $invariant_count | $([ $invariant_count -lt 100 ] && echo 'Slug+link sweep (Phase 2b)' || echo 'Slug sweep + per-slug test linkage') |"
echo "| comment-hygiene | $hygiene_count | $([ $hygiene_count -lt 200 ] && echo 'Tag in one pass' || echo 'Budget ratchet (Phase 3)') |"
echo "| **Total** | **$total** | |"
echo ""

echo "## Plan"
echo ""

cat <<'EOF'
### Pass 1 — Adopt

- Install the plumbline plugin in this project
- Run `/plumbline:affirm`; commit `.claude/rules/plumbline-cheatsheet.md`
- Run `/plumbline:starter .`; review proposed config; save as `.plumbline.json`; commit
- Record the methodology decision in the project's design log (e.g. `decision:coding-style`)
- Verify: `/plumbline:doctor` reports `healthy`

EOF

if [ $source_count -gt 0 ] || [ $symbol_count -gt 0 ]; then
cat <<EOF
### Pass 2a — @source: sweep ($source_count missing-file + $symbol_count missing-symbol)

- Enable \`source_validity: true\` in \`.plumbline.json\`
- Run \`/plumbline:patterns\` to identify path-prefix clusters
- For each cluster, identify the canonical repo-root prefix
- Bulk-sed across the affected files; iterate on stragglers
- Verify clean; commit

EOF
fi

if [ $invariant_count -gt 0 ]; then
cat <<EOF
### Pass 2b — @blessed-invariant: slug sweep ($invariant_count uncovered)

- Enable \`blessed_invariant_test_coverage: true\`
- For each unique prose annotation, run \`/plumbline:slug "<prose>"\` to get a slug
- Rewrite annotations to slug-first form (slug + em-dash + original prose)
- Run \`/plumbline:link-tests\` to find slugs without test references
- Add \`// @blessed-invariant: <slug> — exercised here\` lines in the closest test file per uncovered slug
- Verify clean; commit

EOF
fi

if [ $hygiene_count -ge 200 ]; then
cat <<EOF
### Pass 3 — comment_hygiene ratchet ($hygiene_count violations)

- Temporarily enable \`comment_hygiene: true\`, run \`/plumbline:budget save\`, revert config
- Commit \`.plumbline-budget.json\` + the (still-disabled) config in one commit
- Run \`/plumbline:ci <platform>\`; add the budget-check step to your CI
- Document the migration backlog in the project's CLAUDE.md / readme
- Ongoing: contributors tag comments as they touch them; periodic \`/plumbline:budget save\` ratchets the baseline down
- Eventual: when the count reaches 0, remove the budget file, flip \`comment_hygiene: true\`, commit

EOF
elif [ $hygiene_count -gt 0 ]; then
cat <<EOF
### Pass 3 — comment_hygiene sweep ($hygiene_count violations)

- Enable \`comment_hygiene: true\`
- Run \`/plumbline:patterns\` and \`/plumbline:suggest\` to identify shapes and proposed fixes
- Tag the comments per-shape; revisit ambiguous cases with the user
- Verify clean; commit

EOF
fi

cat <<'EOF'
### Pass 4 — Maintain (structural, ongoing)

- The `PostToolUse` hook is now live — blocks edits that introduce new violations
- CI runs `plumbline .` on every PR
- Periodically run `/plumbline:consolidate` to surface `@source:` mirrors that should collapse

EOF
```

## After the script runs

Surface the emitted plan to the user as a starting point. Adapt to their planner:

- For **ok-planner** users: each `### Pass` above maps directly to a `## Pass N` in a plan file under `.ok-planner/plans/`. Suggest running `/brainstorm` with this plan as the input, or `/write-plan` if the user wants to skip directly to a plan.
- For **other orchestrators**: the structural shape stays the same; the per-pass details translate directly.

The numbers in the table reflect the current state — they will change once Pass 1 lands (the project's `.plumbline.json` will start excluding things the starter's heuristic catches, like generated dirs). Re-run `/plumbline:port` after Pass 1 to refresh the baseline.
