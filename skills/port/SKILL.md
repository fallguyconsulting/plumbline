# /plumbline:port

Write a complete Plumbline port plan for the current project to a file. The plan enumerates every violation across all three checks and the passes needed to make `plumbline .` return clean with all three checks enabled. Output goes to a markdown file — ready to hand off to a planner or to execute directly.

For the full reference on each phase, see `docs/plumbline-porting-guide.md` in the plugin distribution.

## What this does

1. Runs `plumbline doctor` to assess installation state.
2. Runs a temporary all-checks-on lint (atomic enable → audit → restore so `.plumbline.json` is not modified) to count violations per check.
3. Runs `plumbline patterns` to cluster the violations by shape — this is what makes the `comment_hygiene` pass concretely actionable instead of "6,000 of them, good luck."
4. Writes a phase-by-phase plan to `./plumbline-port-plan.md` (or the path passed as the second arg). The plan's goal is **complete**: executing it leaves the lint clean with all three checks on. The budget ratchet, which is a strategy for projects that won't do the full sweep at once, is referenced as a fallback in the porting guide, not in the plan itself.

The plan skips the **Adopt** pass when doctor reports `healthy`. It skips each check's pass when that check's count is zero. The total passes always include **Maintain** as the steady-state description.

## Run

```bash
plumbline_bin="${CLAUDE_PLUGIN_ROOT%/}/bin/plumbline"
guide_path="${CLAUDE_PLUGIN_ROOT%/}/docs/plumbline-porting-guide.md"
target="${1:-.}"
output="${2:-./plumbline-port-plan.md}"

abs_target=$(cd "$target" && pwd)
project=$(basename "$abs_target")

doctor_out=$(node "$plumbline_bin" doctor "$target" 2>&1 || true)
doctor_healthy=false
if echo "$doctor_out" | grep -q "plumbline doctor: healthy"; then
  doctor_healthy=true
fi

# Temporary all-checks-on audit; trap restores .plumbline.json on any exit path
cfg="$target/.plumbline.json"
restore_cfg() {
  if [ -f /tmp/plumbline-port-cfg.bak ]; then
    cp /tmp/plumbline-port-cfg.bak "$cfg" 2>/dev/null || true
    rm -f /tmp/plumbline-port-cfg.bak
  fi
}
trap restore_cfg EXIT INT TERM
if [ -f "$cfg" ]; then
  cp "$cfg" /tmp/plumbline-port-cfg.bak
  sed -i '' 's/"comment_hygiene": false/"comment_hygiene": true/' "$cfg" 2>/dev/null || true
  sed -i '' 's/"source_validity": false/"source_validity": true/' "$cfg" 2>/dev/null || true
  sed -i '' 's/"blessed_invariant_test_coverage": false/"blessed_invariant_test_coverage": true/' "$cfg" 2>/dev/null || true
fi
audit_out=$(node "$plumbline_bin" "$target" 2>&1 || true)
patterns_out=$(node "$plumbline_bin" patterns "$target" 2>&1 || true)
restore_cfg
trap - EXIT INT TERM

source_count=$(echo "$audit_out" | grep -c "plumbline/source-missing-file" || true)
symbol_count=$(echo "$audit_out" | grep -c "plumbline/source-missing-symbol" || true)
invariant_count=$(echo "$audit_out" | grep -c "plumbline/blessed-invariant-uncovered" || true)
hygiene_count=$(echo "$audit_out" | grep -c "plumbline/comment-hygiene" || true)
total=$((source_count + symbol_count + invariant_count + hygiene_count))

{
  echo "# Plumbline port plan — $project"
  echo ""
  echo "Generated by \`/plumbline:port\`."
  echo ""
  echo "**Goal**: execute this plan and \`plumbline .\` returns clean with all three checks (\`source_validity\`, \`blessed_invariant_test_coverage\`, \`comment_hygiene\`) enabled."
  echo ""
  echo "Reference: \`$guide_path\`"
  echo ""
  echo "## Current state"
  echo ""
  echo '```'
  echo "$doctor_out"
  echo '```'
  echo ""
  echo "## Backlog (with all three checks enabled)"
  echo ""
  echo "| Check | Count |"
  echo "| --- | --- |"
  echo "| source-missing-file | $source_count |"
  echo "| source-missing-symbol | $symbol_count |"
  echo "| blessed-invariant-uncovered | $invariant_count |"
  echo "| comment-hygiene | $hygiene_count |"
  echo "| **Total** | **$total** |"
  echo ""

  if [ "$total" -eq 0 ] && [ "$doctor_healthy" = "true" ]; then
    echo "## Result"
    echo ""
    echo "Already Plumbline-clean. Nothing to do."
    echo ""
  else
    echo "## Plan"
    echo ""

    pass=1

    if [ "$doctor_healthy" != "true" ]; then
      cat <<EOF
### Pass $pass — Adopt

- Install the plumbline plugin in this project
- Run \`/plumbline:affirm\`; commit \`.claude/rules/plumbline-cheatsheet.md\`
- Run \`/plumbline:starter .\`; review the proposed config; save as \`.plumbline.json\`; commit
- Record the methodology decision in the project's design/decision log
- Verify: \`/plumbline:doctor\` reports \`healthy\`

EOF
      pass=$((pass + 1))
    fi

    if [ "$source_count" -gt 0 ] || [ "$symbol_count" -gt 0 ]; then
      cat <<EOF
### Pass $pass — \`@source:\` validity ($source_count missing-file + $symbol_count missing-symbol)

Enable \`source_validity: true\` in \`.plumbline.json\`.

EOF
      if [ "$source_count" -gt 0 ]; then
        cat <<'EOF'
**Missing-file violations:** `@source:` references pointing at paths that don't exist. Likely cause: module-relative paths that should be repo-rooted, or files that moved/were deleted. Action: run `/plumbline:patterns` to identify path-prefix clusters; bulk-rewrite each cluster to the correct prefix; delete annotations whose canonical is gone.

EOF
      fi
      if [ "$symbol_count" -gt 0 ]; then
        cat <<'EOF'
**Missing-symbol violations:** `@source:` references whose symbol grep didn't find. Action: triage each — symbol renamed (update annotation), annotation stale (delete), canonical deleted (delete or repoint).

EOF
      fi
      echo "Verify: \`plumbline .\` reports zero violations of these checks. Commit."
      echo ""
      pass=$((pass + 1))
    fi

    if [ "$invariant_count" -gt 0 ]; then
      cat <<EOF
### Pass $pass — \`@blessed-invariant:\` coverage ($invariant_count uncovered)

Enable \`blessed_invariant_test_coverage: true\` in \`.plumbline.json\`.

- For each unique prose annotation, run \`/plumbline:slug "<prose>"\` to get a stable slug
- Rewrite annotations to slug-first form: \`@blessed-invariant: <slug> — <original prose>\`
- Run \`/plumbline:link-tests\` to find slugs without test references
- Add \`// @blessed-invariant: <slug> — exercised here\` lines in the closest test file per uncovered slug

Verify: \`plumbline .\` reports zero violations of this check. Commit.

EOF
      pass=$((pass + 1))
    fi

    if [ "$hygiene_count" -gt 0 ]; then
      cat <<EOF
### Pass $pass — \`comment_hygiene\` ($hygiene_count violations)

Enable \`comment_hygiene: true\` in \`.plumbline.json\`.

Every prose comment must either carry a structured tag (\`@constraint:\`, \`@deliberate:\`, \`@agent-contract\`, \`@blessed-invariant:\`, or a project-extended tag from \`.plumbline.json\`'s \`tags_extend\`) or be deleted. The work is per-comment judgment, but it decomposes by *shape* — the same edit pattern repeats across many sites in each cluster below.

Pattern clusters (from \`plumbline patterns\`):

\`\`\`
EOF
      echo "$patterns_out"
      cat <<'EOF'
```

For each cluster:

- Sample 3-5 violations to confirm the shape
- Decide the standard action for that shape (which tag, or delete as residue)
- Run `/plumbline:suggest` for per-violation proposed fixes within the cluster's files
- Apply and verify the cluster is clean before moving on

Verify: `plumbline .` reports zero violations of this check. Commit.

EOF
      pass=$((pass + 1))
    fi

    cat <<EOF
### Pass $pass — Maintain (steady state)

- The \`PostToolUse\` hook blocks edits that introduce new violations
- Run \`/plumbline:ci <platform>\` and add the emitted workflow to CI
- Periodically run \`/plumbline:consolidate\` to surface \`@source:\` mirrors that should collapse into a shared definition

EOF

    cat <<'EOF'
## Notes for the executor

- Each pass must leave the lint clean for the checks it addresses before the next pass begins. Enable the check, drive its violations to zero, verify, commit.
- The `@source:` and `@blessed-invariant:` passes are mechanical and complete in a single sitting once the patterns are understood.
- The `comment_hygiene` pass is the largest unit of work on a mature codebase; the pattern clusters above make it tractable by reducing N violations to K shapes with a standard action per shape.
- If the project explicitly decides not to do the full `comment_hygiene` sweep at once, the **Ratchet fallback** (`plumbline budget save` + the CI budget-check step) captures a baseline and prevents regression while the backlog burns down opportunistically. The fallback path is documented in the porting guide referenced at the top — it is not part of this plan, because this plan's goal is the clean-state contract.

EOF
  fi
} > "$output"

echo "Wrote port plan: $output"
echo "Backlog: $total total violations across the enabled checks."
```

## After the script runs

The plan is at the path printed by the script (default `./plumbline-port-plan.md`).

Before handing it off, check whether the project uses a plan/spec convention and offer to move or adapt the file accordingly:

- Look for directories like `.ok-planner/`, `.specs/`, `docs/plans/`, `plans/`, `proposals/`, `design/`
- Check the project's `CLAUDE.md` or `README.md` for a documented plan/spec workflow
- If found, ask the user whether to move the file into that convention (matching its naming — date prefix, sequence number, etc.) and whether to reshape headings if the convention expects a specific structure
- If none found, leave the file at the default path

If the project has a planning orchestrator (a `/brainstorm`, `/write-plan`, or similar), suggest invoking it with this file as the input.

The plan's goal — `plumbline .` returns clean with all three checks enabled — is the binding contract for whatever executes it. The plan is **complete** in that sense: it enumerates the work needed to drive every check to zero. The budget-ratchet path (for projects that explicitly will not do the full sweep) is documented in the porting guide, not in the emitted plan.
